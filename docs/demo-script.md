## 关键话术

- 「执行的不是 LLM 的原文，而是 AST 重新渲染的规范化 SQL——注释混淆这类绕过被结构性消除。」
- 「前端的确认按钮只是交互，真正的防线在服务端：执行接口对回传 SQL 重新做完整校验。」
- 「安全假设是任何一层都可能被突破：即使守卫全破，数据库连接本身是只读的，写不进去。」
- 「LLM 只负责翻译问题和修报错，能不能执行由确定性代码决定。」
